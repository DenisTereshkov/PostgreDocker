# Проект: PostgreSQL в Docker + Python подключение через SQLAlchemy

## Описание

Этот проект демонстрирует запуск PostgreSQL в Docker-контейнере с конфигурацией из файла `.env` и подключение к базе из Python скрипта с помощью SQLAlchemy.

---

## Структура проекта

```
.
├── docker-compose.yml      # Конфигурация Docker Compose для PostgreSQL
├── .env                   # Файл с переменными окружения (не коммитить в Git!)
├── requirements.txt       # Файл с зависимостями Python
├── test_app.py            # Python скрипт для теста подключения к БД
└── README.md              # Этот файл
```

---

## Установка и запуск Docker контейнера с postgres.

1. Клонируйте репозиторий.
 в
2. Перейдите в папку `/app` и создайте файл `.env` в соответствии `.env_example`:

3. Запустите PostgreSQL контейнер:

```bash
docker-compose up -d
```

## Тестирование работоспособности контейнера

4. Находясь в корне проекта, создайте вертуальное окружение и установите зависимости Python из `requirements.txt`:

```bash
pip install -r requirements.txt
```

   (Это установит `sqlalchemy`, `psycopg2-binary` и `python-dotenv`.)

5. Запустите тестовый скрипт:

```bash
python app/test_app.py
```

---

## Что делает скрипт `test_app.py`

- Загружает переменные окружения из `.env`.
- Формирует строку подключения к PostgreSQL.
- Подключается к базе и выполняет простой запрос `SELECT 1`.
- Выводит результат и статус подключения.

---

## Важные замечания

- Никогда не коммитьте `.env` в публичные репозитории — там хранятся пароли.
- Для продакшена используйте более безопасные способы хранения секретов.
- Порт 5432 на локальной машине должен быть свободен для проброса порта контейнера.
